# 子 PRD：结束页质量信号与手动纠偏（开始）

| 字段 | 内容 |
|------|------|
| ID | C-05 |
| Tab | 开始 |
| 优先级 | P0 |
| 依赖 | C-02（需更新同一条会话记录或约定「纠偏后写最终值」） |
| 预估 | 中小 |

## 1. 目标

在结束摘要上增强**可信度**：展示**一项**质量信号；允许用户 **+1 / -1** 纠偏，并将**最终次数**写回本地会话，供历史展示。

## 2. 用户故事

作为用户，若最后一次计数明显不准，我希望在结束前微调数字，并知道这次训练「检测质量大概如何」。

## 3. 范围

**In**

- **质量信号（二选一，实现时锁定一种）**  
  - A：`有效检测帧占比` = 有可靠 pose 的帧数 / 总处理帧数；或  
  - B：`稳定度`（基于 Hip Y 方差等已有指标的简化 0～100 分）。  
  在结束摘要上以**一行文案 + 可选简单进度/色点**展示即可。
- **手动纠偏**：`+1` / `-1` 按钮；下限 0；上限可设合理 cap（如 9999）或不做硬 cap。
- 用户点击「确认完成」或等价：将**最终 jumpCount**（及可选 `qualityScore` / `poseCoverage` 字段）**更新到 C-02 已插入的那条记录**；若 C-02 为插入后立刻再更新，需保证事务或顺序一致。
- 若用户不纠偏，质量信号仍写入（若有计算）。

**Out**

- 训练中实时 ±1（可作为增强另开子 PRD；本需求以结束页为主即可闭环）。
- 周报表、导出。

## 4. 验收标准

1. 纠偏后历史（H-01 接上后）看到的次数与摘要最终一致。
2. 质量信号在「无 pose 数据」时有降级展示（如「—」），不崩溃。
3. 仅结束流程内多次 ±1 不产生多条 DB 记录。

## 5. 数据字段建议

若当前 `JumpRecord` 无字段：新增可空列 `adjustedFrom: Int?` 与 `finalJumpCount` 或仅用 `jumpCount` 存最终值并在备注列存 JSON——**选最简单一种**，在 PR 描述中说明。

## 6. 手动测试要点

- 连续 +5 再 -2，保存后重启 App，历史数字不变。
