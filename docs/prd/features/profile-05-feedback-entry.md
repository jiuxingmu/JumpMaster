## 1. 目标

提供低成本的**用户反馈**出口（邮箱或表单 URL），完成「用户能联系到我们」的闭环。

## 2. 用户故事

作为用户，当计数不准或有 bug 时，我希望一键发起反馈而不去商店写差评。

## 3. 范围

**In**

- 设置行「意见反馈」。
- **方案 A**：`mailto:` 带主题模板 `JumpMaster反馈` 与可选 `body` 附 `版本号`、机型信息（注意长度与隐私，仅设备型号与 Android 版本即可）。
- **方案 B**：打开固定 URL（Google Form 等）；与 P-04 同打开方式。

二选一实现即可；另一方案写为后续优化。

**Out**

- 应用内截图标注、客服工单系统。

## 4. 验收标准

1. 点击后进入邮件客户端或浏览器；无可用应用时有 Toast 提示。
2. 不收集未声明的个人信息；邮件正文模板在 PR 中披露字段列表。

## 5. 手动测试要点

- 无邮件 App 的设备（模拟器）行为可接受。
