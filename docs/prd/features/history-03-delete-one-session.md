# 子 PRD：删除单条会话（历史）

| 字段 | 内容 |
|------|------|
| ID | H-03 |
| Tab | 历史 |
| 优先级 | P1 |
| 依赖 | H-01（列表）；若在详情删除则需 H-02 |
| 预估 | 小 |

## 1. 目标

用户可删除**一条**错误或无用的会话记录，并有确认防误触。

## 2. 用户故事

作为用户，我希望删掉点错的记录，以免历史里一直留着脏数据。

## 3. 范围

**In**

- 入口二选一或都要（实现时写清）：列表项滑动删除 / 列表项菜单「删除」/ 详情页删除按钮。推荐：**详情页删除** + 列表**可选**长按菜单（颗粒度小则只做详情页即可闭环）。
- 确认对话框：「删除后无法恢复」+ 取消 / 删除。
- 删除后：返回列表并**刷新**；若从列表菜单删除则该项移除动画或整表刷新。

**Out**

- 清空全部（H-05）、云端同步。

## 4. 验收标准

1. 删除后 DB 中无该 `id`；列表不再出现。
2. 删除中显示 loading 或禁用按钮，防重复点击双删。
3. 删除失败有 Snackbar，列表保持原状。

## 5. 手动测试要点

- 删除最后一条 → 列表为空 → 与 H-04 空状态衔接。
