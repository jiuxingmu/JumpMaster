# 子 PRD：训练中断策略（后台 / 锁屏）（开始）

| ID | C-06 |
| Tab | 开始 |
| 优先级 | P0 |
| 依赖 | C-01 |
| 预估 | 小 |

## 1. 目标

定义并落实**系统中断**时训练与计数的行为，避免用户误以为「还在计」；行为与文案一致、可测。

## 2. 用户故事

作为用户，当我锁屏或切换 App 时，我希望明确知道计数是否暂停，回到 App 时状态一目了然。

## 3. 范围（需产品锁定一种，实现按锁定版本）

**推荐方案 A（默认建议）**

- `onPause` / 等价：自动进入 **Paused**（停止计数增量；相机按 Android 策略释放或暂停分析，与现有 CameraX 生命周期对齐）。
- 回到前台：保持 **Paused**，由用户点击「继续」再 **Active**；显示轻提示「已暂停：离开期间未计数」。

**备选方案 B**

- 进入后台即视为**结束训练**并弹出保存流程——易引发误触，**不推荐**除非有强合规要求。

**In（无论 A/B）**

- 用户可见的 Snackbar / Banner / 状态文案与 `Paused`/`Active` 一致。
- 与 C-01 时长规则一致：后台暂停期间**不计入有效时长**（若选方案 A）。

**Out**

- 前台服务、锁屏后台持续计数（1.x 另立项）。

## 4. 验收标准

1. Home 键切出 10s 再回：计数不偷偷增加（方案 A）。
2. 锁屏再解锁：无崩溃、相机无泄漏（重复进入不 OOM）。
3. 文案与设置中未来「说明」不矛盾（可先写死在 string resource）。

## 5. 手动测试要点

- 来电界面（若可模拟）：行为符合方案 A。
- 低内存杀进程：不要求保留训练中状态（可接受回 Idle）；若保留为加分项，不纳入本需求必达。
