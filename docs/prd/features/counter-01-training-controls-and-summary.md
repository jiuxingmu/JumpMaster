# 子 PRD：训练控制与结束摘要（开始）

## 1. 目标

在不强制要求「已落库」的前提下，打通**单次训练**的交互闭环：用户能**开始 / 暂停 / 结束**，并在结束时看到**次数 + 时长**的明确反馈。

## 2. 用户故事

作为用户，我希望在「开始」页用清晰按钮控制训练节奏，并在结束时立刻看到本次结果，以便确认这一轮是否被正确统计。

## 3. 范围

**In**

- 训练状态至少三态：`Idle`（未开始）、`Active`（计数中）、`Paused`（暂停计数，相机可保持预览或按现有实现冻结计数逻辑）。
- 主按钮语义：`开始` → `暂停` / `继续`；独立 `结束训练`（或等价入口，避免误触）。
- **会话时长**：从首次进入 `Active` 起算，暂停期间不计入「有效训练时长」或计入「总停留时长」二选一，**须在 UI 或子 PRD 备注中写死一种**（推荐：暂停不计入有效时长）。
- 结束：二次确认（对话框即可）→ 展示**结束摘要**（BottomSheet / 全屏 Dialog / 独立 composable 均可）：**本次次数**、**有效时长**。
- 摘要关闭后回到 `Idle`，计数归零或明确「新会话」起点（与 C-02 衔接时由实现选定一种并文档化）。

**Out**

- Room 写入、历史列表（见 C-02）。
- 权限引导页（见 C-03）、画面质量条（见 C-04）、质量百分比（见 C-05）。

## 4. 验收标准

1. 从 `Idle` 经 `Active` 到结束摘要，全程无崩溃；重复进入退出 3 次行为一致。
2. 暂停期间计数器**不增加**（与现有 `CameraJumpDetector` / `JumpCounter` 行为对齐）。
3. 结束摘要上的**次数**与训练页实时数字一致；**时长**与秒表误差 ≤1s（肉眼或日志可对）。
4. 无持久化情况下，杀进程后数据丢失为**可接受**；不阻塞 C-02。

## 5. 手动测试要点

- 仅开始不结束 → 返回再进，状态符合预期（建议回到 Idle，若有争议写进实现说明）。
- 暂停 → 继续 → 结束，时长与次数合理。
- 旋转屏幕（若支持）：摘要数字不丢不乱。

## 6. 实现提示（非规范）

状态放在 `MainViewModel` 的 `StateFlow`；Compose 无状态展示。与 `docs/prd.md` 第 4.1 节「开始/暂停/结束」对齐。
