# 子 PRD：计数灵敏度档位（我的）

## 1. 目标

提供 **低 / 中 / 高**（或等价三档）灵敏度，影响视觉计数阈值或滤波强度，持久化并在**新开始的训练**中生效。

## 2. 用户故事

作为用户，我希望在容易误计或漏计时能调一档，不用懂算法参数。

## 3. 范围

**In**

- UI：`SegmentedButton` / `ExposedDropdownMenu` / Radio 三选一，展示档位名称与一句说明（如：高更灵敏、低更稳）。
- 持久化；ViewModel 或单例配置在会话 `Active` 开始时读取当前档位映射到域内参数（阈值 A/B、滤波窗口等，由研发列映射表）。
- 训练中切换档位：**推荐下一会话生效**，避免中途改变破坏可比性；若必须热更需在 PR 论证。

**Out**

- 按身高体重自动推荐档位。
- 传感器模式参数。

## 4. 验收标准

1. 三档可切换且重启后保持。
2. 至少有一种可观测差异（日志或演示视频）：例如高档对小幅动作更敏感（在测试脚本或手动场景下）。
3. 与 C-04 提示文案不矛盾（若高档更易抖，可在说明中一句提示）。

## 5. 手动测试要点

- 默认档与旧版行为一致（回归）。
