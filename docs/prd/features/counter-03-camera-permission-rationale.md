## 1. 目标

在申请 `CAMERA` 前给出**场景化说明**，在拒绝后提供**可理解的空态**与**跳转系统设置**入口，形成权限闭环。

## 2. 用户故事

作为用户，我希望知道为什么需要相机，并在误拒权限后仍能自行打开权限继续使用。

## 3. 范围

**In**

- 首次进入「开始」且未授权：展示简短说明（1～2 句）+ 主按钮「允许使用相机」→ 触发系统请求。
- 已拒绝且「不再询问」或等价：展示说明 +「去设置开启」→ `Intent` 打开应用详情页。
- 已授权：直接进入现有预览流程，无多余挡板。
- 拒绝但未永久拒绝：可再次点击重试请求（避免死循环弹窗）。

**Out**

- 麦克风、通知权限。
- 首次摆放图文引导（C-07）。

## 4. 验收标准

1. 授权前后布局不崩溃；从设置返回后权限状态**自动刷新**（`Lifecycle` / `DisposableEffect` 等）。
2. 文案不出现「监控」「录像上传」等易误解表述；与主 PRD 合规方向一致。
3. 无相机硬件的模拟器：有降级提示或不崩溃（按团队策略二选一写死）。

## 5. 手动测试要点

- 允许 → 拒绝 → 去设置 → 打开，各路径走一遍。
- 深色模式下图标与文字对比度可读。
