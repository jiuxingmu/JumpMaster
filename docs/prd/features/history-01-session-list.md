# 子 PRD：历史会话列表（历史）

| 字段 | 内容 |
|------|------|
| ID | H-01 |
| Tab | 历史 |
| 优先级 | P1 |
| 依赖 | C-02（本地已有会话数据） |
| 预估 | 小 |

## 1. 目标

在「历史」Tab 展示按时间倒序的**会话列表**，每条显示用户最关心的信息，完成「练完能回看」的读路径闭环。

## 2. 用户故事

作为用户，我希望打开历史就能看到最近几次跳绳记录，以便确认今天练了多少。

## 3. 范围

**In**

- 从 Repository / ViewModel 读取会话列表（Flow 或 paging 皆可，数据量小可用简单列表）。
- 每行至少：**日期时间**（或「今天 20:35」类相对时间）、**次数**、**时长**。
- 下拉刷新可选；无则进入页面自动加载即可。
- 点击进入详情（H-02 实现前可先 `Toast` 占位，**推荐**：本需求一并做「点击行」导航占位路由，避免半成品——若 H-02 同迭代可做 `navigate` 到 detail 空壳）。

**Out**

- 详情内容（H-02）、删除（H-03）、按周聚合图表。

## 4. 验收标准

1. C-02 保存的会话在列表中可见，顺序为新在上。
2. 列表为空时不崩溃（空 UI 见 H-04，可先简单 Text「暂无记录」占位，H-04 再换设计）。
3. 滑动流畅（本地小数据集无卡顿）。

## 5. 手动测试要点

- 保存 10 条后滚动与旋转屏幕。
- 时区变更（可选）：日期显示合理。

## 6. 实现提示

列表项 Composable 可抽到 `HistorySessionRecordRows.kt` 或现有文件，遵守单文件行数规范。
