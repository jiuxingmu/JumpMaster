# 子 PRD：会话详情页（历史）

| 字段 | 内容 |
|------|------|
| ID | H-02 |
| Tab | 历史 |
| 优先级 | P1 |
| 依赖 | H-01 |
| 预估 | 小 |

## 1. 目标

用户从列表点进某次训练，看到**完整一次会话**的信息，并可返回列表。

## 2. 用户故事

作为用户，我想点开某条记录查看具体次数、时长和开始结束时间。

## 3. 范围

**In**

- 导航：列表行 → 详情（Compose destination 或独立 Screen，与现有 `HistoryScreen` 结构一致即可）。
- 详情字段：**最终次数**、**时长**、**开始时间**、**结束时间**（或起止缺一则用已有字段推导）；若 C-05 已写入质量信号，一并展示。
- 顶部返回（系统返回键行为正确）。

**Out**

- 编辑备注、分享图片。

## 4. 验收标准

1. 非法 `sessionId`（深链模拟）不崩溃，展示「记录不存在」或自动 pop。
2. 展示数据与 DB 一致。
3. 详情页不直接访问 Dao，经 ViewModel。

## 5. 手动测试要点

- 从列表进入 → 返回 → 再进另一条。
- 删除该条后（H-03）从返回栈回到列表是否刷新（应在 H-03 定义；本需求至少保证再次进入列表时数据正确）。
