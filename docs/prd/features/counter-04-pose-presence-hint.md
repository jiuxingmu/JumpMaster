# 子 PRD：人体检测可见性提示（开始）

| 预估 | 小 |

## 1. 目标

在预览过程中，当**长时间未检测到可靠人体姿态**时，给出轻量提示，帮助用户调整站位与光线，降低「乱跳/不计」的挫败感。

## 2. 用户故事

作为用户，当手机看不到我时，我希望屏幕上明确提示，而不是沉默失败。

## 3. 范围

**In**

- 基于现有帧处理结果定义简单规则（示例，实现可调整）：连续 N 帧无有效 landmarks 或置信度低于阈值 → 显示一条**非阻塞**提示（Snackbar / 顶栏文案 / chip 三选一，全 App 统一一种）。
- 恢复检测到人体后，提示**自动消失**（可带短延迟防抖）。
- 提示文案 1～2 条轮换或单条即可，例如：「请让全身进入画面」「再后退一步试试」。

**Out**

- 多人同框策略、灵敏度算法改动（可另开需求）。
- 结束页质量百分比（C-05）。

## 4. 验收标准

1. 用手遮挡镜头 / 离开画面 → 在约定时间内出现提示；回到画面中提示消失。
2. 提示不遮挡主计数数字（或仅短暂遮挡，需说明）。
3. 不误报过于频繁：同一状态切换有防抖（例如 ≥1s 间隔），具体参数写在代码常量处。

## 5. 手动测试要点

- 弱光环境：提示是否仍合理（允许「弱光」类文案若已实现测光，否则不单开测光）。
